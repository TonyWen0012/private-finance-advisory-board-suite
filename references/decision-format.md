# Decision Format

## Standard answer shape

1. Decision type
2. Personas used
3. Current state summary
4. Recommendation
5. Risks and tradeoffs
6. Next action
7. Memory writeback

## Recommendation labels

- `do`
- `do_not`
- `split`
- `delay`
- `needs_more_data`

## Minimum evidence standard

Do not give a high-confidence answer unless at least one of these is available:
- current cash flow
- debt details
- time horizon
- expected return
- risk tolerance
- decision deadline

## Memory writeback format

- question type
- recommendation label
- key facts used
- assumptions
- risks
- follow-up date

## If data is missing

- say what is missing
- say which persona needs it
- ask for only the smallest necessary missing facts
